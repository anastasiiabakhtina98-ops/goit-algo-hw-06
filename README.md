# Висновки

У роботі змодельовано транспортну мережу напрямку руху в місті Ужгород з вул. Берчені до пл. Петефі з двома альтернативними маршрутами: через міст на вул. Шпеника та через міст на вул. Масарика. Граф реалізовано як орієнтований, де вершини відповідають ключовим ділянкам маршруту, а ребра — можливим напрямкам руху автомобіля між цими ділянками.

Для цього графа було реалізовано два алгоритми пошуку шляху: DFS (Depth-First Search) та BFS (Breadth-First Search). Обидва алгоритми знайшли коректні маршрути від вершини «Берчені» до вершини «Площа_Петефі», причому довжина обох шляхів виявилася однаковою — 5 ребер.

**Алгоритм DFS** повернув шлях через міст Масарика:  
`Берчені -> Кільце_Берчені -> Шлях_до_Масарика -> Міст_Масарика -> Правий_берег_центр -> Площа_Петефі`.  
Це пов’язано з тим, що DFS заглиблюється в одну гілку графа доти, доки може рухатися вперед, і порядок обходу визначається порядком сусідів у структурі даних (у даній реалізації спочатку розглядається гілка на Масарика).

**Алгоритм BFS** повернув шлях через міст Шпеника:  
`Берчені -> Кільце_Берчені -> Шлях_до_Шпеника -> Міст_Шпеника -> Правий_берег_центр -> Площа_Петефі`.  
BFS обходить граф пошарово, розглядаючи спочатку всі вершини на відстані 1 ребра, потім 2 тощо, тому він знаходить найкоротший шлях за кількістю ребер; у нашій моделі обидва маршрути мають однакову довжину, тому BFS повертає перший за порядком огляду варіант.

Отже, на цьому прикладі видно, що DFS не гарантує знаходження «оптимального» маршруту, а лише першого знайденого шляху з урахуванням порядку сусідів, тоді як BFS завжди знаходить шлях мінімальної довжини та є більш придатним для задач пошуку найкороткого маршруту в транспортних мережах. Водночас обидва алгоритми коректно працюють на побудованому графі й демонструють різну стратегію обходу однієї й тієї ж моделі міської дорожньої мережі.
